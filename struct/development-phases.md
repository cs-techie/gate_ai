# Development Phases

This document breaks the project into phases, estimated durations, and primary milestones.

## Phase 0: Planning (1 week)

- Finalize PRD, TRD, Architecture
- Define MVP scope and acceptance criteria

## Phase 1: Core Backend (2-3 weeks)

- Implement auth, users, tests, questions, results
- Implement DB models and migrations
- Implement scoring engine
- Unit tests for core logic

## Phase 2: Admin and Materials (1-2 weeks)

- Admin endpoints for create-test and add-question
- File upload, storage, and validation
- Material listing and viewing

## Phase 3: Frontend MVP (2-3 weeks)

- Implement login/signup pages
- Student dashboard, test runner, result page
- Admin pages for test creation and material upload

## Phase 4: Integrations and Deployment (1-2 weeks)

- Dockerize services
- CI pipeline
- Deploy to cloud (single server)

## Phase 5: Testing and Stabilization (1 week)

- Integration tests, end-to-end smoke tests
- Security checks, vulnerability scans
- Performance tuning

## Phase 6: V2 Planning and Handoff (1 week)

- Plan analytics, charts, and AI services
- Backlog grooming and roadmap

---

Notes:
- Estimates assume a small team (1-3 engineers).
- Timelines may shift based on scope changes.
